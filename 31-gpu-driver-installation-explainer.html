GPU Driver Installation Overview

GPU drivers are essential software components that enable communication between the operating system and GPU hardware. Without proper drivers, GPUs cannot function, and applications cannot access GPU resources. The installation process involves several critical steps: verifying system compatibility, preparing the environment, installing the driver package, and verifying successful installation.

1Hardware Verification

Verify that NVIDIA GPU hardware is present and detected by the system:

lspci | grep -i nvidia

This command lists all NVIDIA devices connected to the system. You should see entries for your GPU(s).

2Operating System Check

Verify your operating system version and kernel:

uname -a
cat /etc/os-release

Ensure your OS version is supported by the NVIDIA driver. Check NVIDIA's compatibility matrix for supported distributions.

3Disk Space Requirements

Ensure sufficient disk space is available:

  • Driver package: ~300-500 MB
  • Installation files: ~1-2 GB
  • CUDA toolkit (if installing): ~3-5 GB

Pre-Installation Steps

Proper preparation before driver installation prevents conflicts and ensures a smooth installation process. These steps include removing old drivers, disabling conflicting services, and preparing the system environment.

Remove Existing Drivers

If previous NVIDIA drivers are installed, remove them first:

sudo apt-get remove --purge '^nvidia-.*'
sudo apt-get autoremove

For RPM-based systems:

sudo yum remove nvidia-*
sudo dnf remove nvidia-*

Disable Nouveau Driver

The open-source Nouveau driver can conflict with NVIDIA drivers. Disable it:

sudo bash -c "echo blacklist nouveau > /etc/modprobe.d/blacklist-nvidia-nouveau.conf"
sudo bash -c "echo options nouveau modeset=0 >> /etc/modprobe.d/blacklist-nvidia-nouveau.conf"

Then regenerate the initramfs:

sudo update-initramfs -u

Install Build Dependencies

Install required build tools and kernel headers:

sudo apt-get update
sudo apt-get install build-essential dkms linux-headers-$(uname -r)

Driver Installation Process

The driver installation can be performed using package managers or by downloading the driver directly from NVIDIA. Package manager installation is recommended for most users as it handles dependencies automatically.

Method 1: Package Manager Installation (Recommended)

For Ubuntu/Debian systems:

sudo apt-get update
sudo apt-get install nvidia-driver-535
# Or use the latest version available

For RHEL/CentOS/Fedora:

sudo dnf install nvidia-driver

Method 2: Direct Driver Download

Download the driver from NVIDIA's website and install:

# Download driver (example for Linux x86_64)
wget https://us.download.nvidia.com/XFree86/Linux-x86_64/535.XX.XX/NVIDIA-Linux-x86_64-535.XX.XX.run

# Make executable
chmod +x NVIDIA-Linux-x86_64-535.XX.XX.run

# Install
sudo ./NVIDIA-Linux-x86_64-535.XX.XX.run

Installation Options

During installation, you may be prompted for:

  • 32-bit compatibility libraries: Required for some legacy applications
  • DKMS support: Automatic driver rebuilds when kernel updates
  • X server configuration: Automatic X.org configuration

Verification and Testing

After installation, verify that the driver is properly installed and the GPU is functioning correctly. This involves checking driver version, GPU detection, and basic functionality tests.

Check Driver Version

nvidia-smi

This command displays GPU information, driver version, CUDA version, and GPU utilization. If this command works, the driver is installed correctly.

Verify GPU Detection

nvidia-smi --query-gpu=name,driver_version,memory.total --format=csv

This provides detailed information about detected GPUs, confirming hardware recognition.

Test CUDA Functionality

If CUDA toolkit is installed, test basic CUDA operations:

nvcc --version
# Run a simple CUDA test program if available

Troubleshooting Common Issues

Driver installation can encounter various issues. Understanding common problems and their solutions helps resolve installation failures quickly.

Issue: Driver Installation Fails

Symptoms: Installation script fails or returns errors

Solutions:

  • Check kernel headers are installed: uname -r matches installed headers
  • Verify sufficient disk space
  • Check installation logs: /var/log/nvidia-installer.log
  • Ensure secure boot is disabled (if applicable)

Issue: GPU Not Detected After Installation

Symptoms: nvidia-smi shows "No devices found"

Solutions:

  • Verify GPU is properly seated: lspci | grep -i nvidia
  • Check driver is loaded: lsmod | grep nvidia
  • Reboot the system
  • Verify PCIe power connections

Issue: X Server Conflicts

Symptoms: Display issues or X server won't start

Solutions:

  • Install in runlevel 3 (text mode): sudo init 3
  • Use --no-x-check flag during installation
  • Manually configure X.org if needed
Best Practices: Always check NVIDIA's official documentation for your specific GPU model and driver version. Keep installation logs for troubleshooting. Test GPU functionality after installation before deploying production workloads. Consider using package managers for easier updates and dependency management.
