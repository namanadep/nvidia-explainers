Storage Commands Overview

Storage management commands are essential tools for managing storage in AI infrastructure. Commands like `df`, `fdisk`, and `mount` enable administrators to monitor storage usage, partition disks, and mount filesystems. Understanding these commands is crucial for storage management, troubleshooting, and optimization in AI clusters. These commands provide the foundation for storage operations, from initial disk setup to ongoing monitoring and maintenance.

df Command

The `df` command displays filesystem disk space usage, showing how much space is used and available on mounted filesystems. This is essential for monitoring storage in AI clusters.

Basic Usage

# Show filesystem usage
df

# Human-readable format
df -h

# Show inodes
df -i

# Show specific filesystem
df -h /data

fdisk Command

The `fdisk` command is used for disk partitioning, creating and managing disk partitions. This is essential for initial disk setup and storage configuration.

Basic Usage

# List partitions
fdisk -l

# Partition a disk
sudo fdisk /dev/sdb

# Interactive commands:
# n - new partition
# d - delete partition
# p - print partition table
# w - write and exit
# q - quit without saving

Partition Types

  • Primary: Primary partitions (up to 4)
  • Extended: Extended partition
  • Logical: Logical partitions within extended
  • GPT: GPT partitions (modern systems)

mount Command

The `mount` command attaches filesystems to the directory tree, making storage accessible. This is essential for accessing storage devices and network filesystems.

graph LR A[Storage Device] --> B[mount] B --> C[Directory Tree] C --> D[Accessible Files] style A fill:#1976D2 style B fill:#2196F3 style C fill:#64B5F6

Basic Usage

# Mount a device
sudo mount /dev/sdb1 /mnt/data

# Mount with options
sudo mount -t ext4 -o rw /dev/sdb1 /mnt/data

# Mount NFS
sudo mount -t nfs server:/data /mnt/data

# Show mounted filesystems
mount
mount | grep nfs

Common Options

  • -t: Specify filesystem type
  • -o: Mount options (rw, ro, noexec, etc.)
  • -a: Mount all filesystems in /etc/fstab
  • -r: Mount read-only
  • -w: Mount read-write

/etc/fstab Configuration

Configure automatic mounting:

# Add to /etc/fstab
/dev/sdb1 /mnt/data ext4 defaults 0 2
server:/data /mnt/data nfs defaults 0 0

# Test fstab
sudo mount -a

# Format: device mountpoint fstype options dump pass

Storage Troubleshooting

Storage troubleshooting involves diagnosing and resolving storage issues using various commands and techniques.

Common Issues

  • Full Filesystem: Filesystem full (use df)
  • Mount Failures: Mount failures (check logs)
  • Permission Issues: Permission issues (check permissions)
  • Device Not Found: Device not found (check lsblk)

Troubleshooting Commands

# Check disk usage
df -h

# List block devices
lsblk

# Check filesystem
sudo fsck /dev/sdb1

# Check mount status
mount | grep /data

# Check disk I/O
iostat -x 1

Additional Commands

  • lsblk: List block devices
  • blkid: Show block device attributes
  • parted: Advanced partitioning
  • du: Show directory disk usage
  • iostat: Monitor disk I/O
Storage Management Mastery: Master storage commands by regularly using `df` to monitor usage, `fdisk` for partitioning (when needed), and `mount` for filesystem management. Use `/etc/fstab` for persistent mounts, and learn additional commands like `lsblk`, `blkid`, and `du` for comprehensive storage management. These commands form the foundation for effective storage management in AI infrastructure.
