Network Performance Tools

Network performance tools measure bandwidth, latency, and network utilization. These tools are essential for analyzing network performance.

iperf

Measure network bandwidth:

iperf Usage:

# Server side iperf3 -s # Client side - TCP test iperf3 -c [server-ip] -t 60 # Client side - UDP test iperf3 -c [server-ip] -u -b 10G -t 60 # Bidirectional test iperf3 -c [server-ip] --bidir # Parallel streams iperf3 -c [server-ip] -P 8

netperf

Comprehensive network performance testing:

netperf Usage:

# Start netserver netserver # TCP stream test netperf -H [server-ip] -t TCP_STREAM # UDP stream test netperf -H [server-ip] -t UDP_STREAM # Request/response test netperf -H [server-ip] -t TCP_RR

InfiniBand Tools

InfiniBand-specific performance tools:

InfiniBand Performance:

# Bandwidth test ib_write_bw [options] # Latency test ib_write_lat [options] # Query performance perfquery [device] # Check link status iblinkinfo

GPU Performance Tools

GPU performance tools monitor GPU utilization, memory usage, and performance metrics. These tools are essential for GPU performance analysis.

nvidia-smi

Monitor GPU status and performance:

nvidia-smi Usage:

# Basic GPU information nvidia-smi # Continuous monitoring nvidia-smi -l 1 # Query specific metrics nvidia-smi --query-gpu=index,name,utilization.gpu,memory.used,memory.total \ --format=csv # Monitor power consumption nvidia-smi --query-gpu=index,power.draw,power.limit --format=csv # Monitor temperature nvidia-smi --query-gpu=index,temperature.gpu --format=csv

Nsight Tools

NVIDIA Nsight tools for profiling:

  • Nsight Systems: System-wide performance analysis
  • Nsight Compute: CUDA kernel profiling
  • Nsight Graphics: Graphics API profiling

Nsight Systems Usage:

# Profile application nsys profile [application] # Profile with specific options nsys profile --trace=cuda,nvtx,osrt [application] # Generate report nsys-ui [profile-file].qdrep

DCGM (Data Center GPU Manager)

Data center GPU monitoring:

DCGM Usage:

# Start DCGM daemon dcgmi dmon -e 155,150,203,204,210,211 -d 1 # Monitor GPUs dcgmi dmon -g [gpu-id] -e 155,150,203,204 # Health check dcgmi diag -r 1

System Performance Tools

System performance tools monitor CPU, memory, I/O, and overall system performance. These tools provide system-wide performance insights.

top and htop

Monitor system processes and resources:

Usage:

# Basic top top # htop (more user-friendly) htop # Sort by CPU usage top -o %CPU # Sort by memory usage top -o %MEM

iostat

Monitor I/O statistics:

Usage:

# Basic I/O statistics iostat # Continuous monitoring iostat -x 1 # Show specific device iostat -x /dev/sda 1

perf

Linux performance analysis tool:

Usage:

# Record performance data perf record [application] # Generate report perf report # Real-time monitoring perf top # Stat summary perf stat [application]

Tool Selection

Select appropriate tools based on analysis requirements. Tool selection depends on what needs to be measured and analyzed.

Selection Criteria

  • Component: Network, GPU, or system
  • Metrics: What needs to be measured
  • Accuracy: Required measurement accuracy
  • Overhead: Tool overhead on system
  • Integration: Integration with monitoring systems

Tool Usage Best Practices

Follow best practices for effective tool usage.

Best Practices

  • Select appropriate tools for each component
  • Understand tool capabilities and limitations
  • Calibrate tools before use
  • Run tests under realistic conditions
  • Document tool usage and results
  • Integrate tools with monitoring systems
Tool Usage Checklist: 1. Identify performance analysis needs
2. Select appropriate tools
3. Install and configure tools
4. Calibrate tools
5. Run performance tests
6. Analyze results
7. Document findings
8. Integrate with monitoring
9. Maintain tool updates
10. Review tool effectiveness
