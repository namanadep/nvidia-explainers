Network Traffic Patterns Overview

AI data centers exhibit two distinct network traffic patterns with fundamentally different characteristics and requirements. East-West traffic refers to communication between servers within the data center, while North-South traffic refers to communication between the data center and external networks. Understanding these patterns is essential for designing appropriate network architectures that can efficiently handle both types of traffic.

East-West Traffic (AI Fabric)

East-West traffic represents horizontal communication between servers within the data center, particularly critical for distributed AI training workloads. This traffic pattern is characterized by massive bandwidth requirements, ultra-low latency needs, and parallel communication patterns as GPUs exchange gradients and model parameters during training.

East-West Traffic

Pattern: Server-to-server (horizontal)

Purpose: GPU-to-GPU communication for training

Characteristics:

  • Terabits per second bandwidth
  • Microsecond latency
  • Predictable, parallel patterns
  • All-to-all communication

Critical for distributed AI training workloads requiring high bandwidth and low latency.

North-South Traffic

Pattern: External-to-data center (vertical)

Purpose: User access and API calls

Characteristics:

  • Gigabits per second bandwidth
  • Millisecond latency
  • Variable, user-driven patterns
  • Client-server communication

Standard networking for user access, API calls, and system management.
