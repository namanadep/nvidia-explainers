Adaptive Routing Overview

Adaptive Routing is an InfiniBand technology that enables dynamic path selection based on real-time network conditions. Unlike static routing that uses predetermined paths, adaptive routing continuously monitors network state and selects optimal paths to avoid congestion, balance load, and optimize performance. This intelligent pathfinding capability is essential for maintaining high performance in large-scale networks where traffic patterns are dynamic and congestion can occur unpredictably.

Adaptive Routing Mechanism

Adaptive routing operates by continuously monitoring network conditions and making dynamic routing decisions at each switch. This enables packets to take different paths based on current network state, optimizing performance and avoiding congestion.

Path Selection Process

  1. Network Monitoring: Switches monitor link utilization and congestion
  2. Path Evaluation: Evaluate multiple available paths
  3. Condition Assessment: Assess congestion and load on each path
  4. Path Selection: Select optimal path based on conditions
  5. Dynamic Adjustment: Continuously adjust as conditions change

Routing Decision Factors

  • Link Utilization: Current load on links
  • Congestion Indicators: Buffer occupancy, queue depth
  • Path Length: Number of hops to destination
  • Historical Performance: Past performance of paths
  • Load Distribution: Balance traffic across paths

Per-Packet Routing

  • Dynamic Selection: Path selected per packet or flow
  • Real-Time Adaptation: Adapts to current conditions
  • Load Balancing: Distributes traffic across paths
  • Congestion Avoidance: Avoids congested paths

Load Balancing

Adaptive routing enables intelligent load balancing by distributing traffic across multiple available paths. This prevents any single path from becoming overloaded and optimizes overall network utilization.

Congestion Avoidance

Adaptive routing proactively avoids congestion by detecting congestion indicators and routing traffic around congested areas. This prevents performance degradation and maintains optimal network operation.

Congestion Detection

  • Buffer Occupancy: Monitor buffer fill levels
  • Queue Depth: Track queue lengths
  • Link Utilization: Monitor link usage
  • Latency Increase: Detect increased latency

Congestion Avoidance Strategies

  • Path Diversion: Route around congested areas
  • Load Redistribution: Redistribute load to less-congested paths
  • Proactive Routing: Anticipate and avoid congestion
  • Dynamic Adjustment: Continuously adjust routes

Performance Impact: In large-scale distributed training, adaptive routing can maintain optimal performance even when network conditions change. Without adaptive routing, congestion on a single path can degrade performance for all traffic using that path. Adaptive routing automatically routes around congestion, maintaining high performance and ensuring consistent training throughput.
