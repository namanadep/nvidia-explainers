BIOS/UEFI Configuration Overview

BIOS (Basic Input/Output System) and UEFI (Unified Extensible Firmware Interface) firmware settings are critical for proper GPU initialization, performance optimization, and system stability. Correct BIOS/UEFI configuration ensures GPUs are detected during POST, initialized with optimal settings, and function correctly in the operating system. Incorrect settings can prevent GPU detection, limit performance, or cause system instability.

UEFI vs Legacy BIOS

Modern servers use UEFI firmware, which replaced legacy BIOS. Understanding the differences helps configure systems correctly for GPU operation.

Legacy BIOS

  • Limitations: 16-bit code, limited to 2.2TB disk support, slower boot
  • GPU Support: Basic GPU initialization, limited multi-GPU support
  • Configuration: Text-based interface, limited options
  • Use Cases: Legacy systems, older hardware

UEFI Firmware

  • Advantages: 32/64-bit code, GPT disk support, faster boot, secure boot
  • GPU Support: Better multi-GPU support, improved initialization
  • Configuration: Graphical interface, extensive options, mouse support
  • Use Cases: Modern servers, recommended for GPU servers

PCIe Configuration Settings

PCIe settings in BIOS/UEFI control how PCIe devices (including GPUs) are detected, initialized, and configured. These settings are critical for multi-GPU systems.

Above 4G Decoding / Above 4GB MMIO

Purpose: Enables memory-mapped I/O (MMIO) above 4GB address space, required for systems with multiple GPUs or large GPU memory.

Location: Advanced → PCIe/PCI Subsystem Settings

Setting: Enable / Enabled

Critical for:

  • Systems with 2+ GPUs
  • GPUs with large memory (40GB+)
  • Multi-GPU training workloads
Why It Matters: Without Above 4G Decoding enabled, systems with multiple GPUs may only detect the first GPU. This setting allows the system to map GPU memory above the 4GB boundary, enabling support for multiple high-memory GPUs.

PCIe Link Speed

Purpose: Sets the PCIe generation (Gen1, Gen2, Gen3, Gen4, Gen5) for PCIe slots.

Location: Advanced → PCIe/PCI Subsystem Settings

Setting: Auto (recommended) or specific generation

Considerations:

  • Auto allows negotiation to highest supported speed
  • Forcing lower speed can limit GPU performance
  • Gen4/Gen5 recommended for modern GPUs (A100, H100)

PCIe Slot Configuration

Purpose: Configures individual PCIe slots (enable/disable, link width, bifurcation).

Location: Advanced → PCIe/PCI Subsystem Settings → Slot Configuration

Settings:

  • Enable/disable specific slots
  • Configure link width (x16, x8, x4)
  • Set bifurcation mode (x16, x8x8, x4x4x4x4)

PCIe ASPM (Active State Power Management)

Purpose: Enables power management for PCIe devices when idle.

Location: Advanced → PCIe/PCI Subsystem Settings

Setting: Disabled (recommended for GPUs)

Why Disable:

  • ASPM can cause GPU initialization delays
  • May cause GPU detection issues
  • GPUs have their own power management

GPU-Specific Settings

GPU-specific BIOS/UEFI settings control GPU detection, initialization, and memory mapping. These settings ensure GPUs are properly recognized and configured.

GPU Detection and Initialization

Settings to Verify:

  • Primary Display: Set to PCIe/PCI (not onboard graphics)
  • GPU Initialization: Enable all GPU slots
  • GPU Memory Mapping: Ensure sufficient address space
  • VGA Priority: Set to PCIe for GPU-first boot

Memory Remapping

Purpose: Remaps memory above 4GB to enable large memory support and GPU memory mapping.

Location: Advanced → Memory Configuration

Setting: Enable

Required for:

  • Systems with 32GB+ system memory
  • Multi-GPU configurations
  • GPUs with large memory (40GB+)

IOMMU / VT-d

Purpose: Enables I/O Memory Management Unit for device passthrough and virtualization.

Location: Advanced → CPU Configuration → Intel VT-d / AMD-Vi

Setting: Enable (if using GPU passthrough or virtualization)

Use Cases:

  • GPU passthrough to VMs
  • vGPU configurations
  • Container GPU access

Boot Configuration

Boot settings control how the system starts and which devices are used for booting. Proper boot configuration ensures reliable system startup and GPU initialization.

Boot Mode

Options: Legacy BIOS, UEFI, or UEFI with CSM (Compatibility Support Module)

Recommendation: UEFI (pure UEFI, no CSM)

Benefits:

  • Faster boot times
  • Better GPU support
  • Secure boot capability
  • GPT disk support

Secure Boot

Purpose: Validates bootloader and driver signatures for security.

Location: Security → Secure Boot

Setting: Disable (for GPU driver compatibility) or Enable with proper keys

Considerations:

  • Secure Boot can prevent unsigned GPU drivers from loading
  • May require driver signing or Secure Boot keys
  • Disable if experiencing GPU driver loading issues

Fast Boot / Quick Boot

Purpose: Skips certain POST checks to speed up boot.

Setting: Disable (recommended for GPU servers)

Why Disable:

  • Ensures full GPU detection during POST
  • Prevents missed hardware initialization
  • Allows proper error reporting

Power Management Settings

Power management settings in BIOS/UEFI control system power behavior, which affects GPU power delivery and thermal management.

Power Management

Settings:

  • ACPI Settings: Enable ACPI for proper power management
  • CPU Power Management: Enable C-states and P-states
  • PCIe Power Management: Disable for GPUs (use GPU power management)

Power Limit Settings

Purpose: Sets maximum power consumption limits for the system.

Considerations:

  • Ensure power limit accommodates all GPUs
  • Account for peak power consumption
  • Consider PSU capacity and efficiency
  • Set appropriate limits to prevent overload

Thermal Management

Settings:

  • Fan Control: Enable intelligent fan control
  • Thermal Throttling: Configure thermal limits
  • Temperature Monitoring: Enable temperature sensors

Configuration Best Practices

Following best practices ensures reliable GPU operation and optimal performance. Document configurations for consistency and troubleshooting.

Essential Settings Checklist

  • ✓ Enable UEFI boot mode (disable Legacy BIOS)
  • ✓ Enable Above 4G Decoding / Above 4GB MMIO
  • ✓ Set PCIe link speed to Auto
  • ✓ Disable PCIe ASPM
  • ✓ Set primary display to PCIe
  • ✓ Enable memory remapping
  • ✓ Disable Fast Boot
  • ✓ Configure secure boot appropriately
  • ✓ Verify all GPU slots are enabled
  • ✓ Set appropriate power limits

Verification Steps

After BIOS/UEFI configuration, verify GPU detection:

# Check GPU detection in system lspci | grep -i nvidia # Verify GPU appears in BIOS/UEFI POST # Check POST messages during boot # Verify GPU in operating system nvidia-smi # Check PCIe link status nvidia-smi -q -d PCIE

Common Issues and Solutions

Issue Likely Cause Solution
GPU not detected Above 4G Decoding disabled Enable Above 4G Decoding
Only first GPU detected Memory remapping disabled Enable memory remapping
Slow GPU performance PCIe link speed limited Set PCIe to Auto or Gen4/Gen5
GPU driver won't load Secure Boot enabled Disable Secure Boot or sign drivers
System instability Power limit too low Increase power limit or reduce GPUs
