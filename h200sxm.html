H200SXM GPU Architecture

The H200 SXM GPU is built on the Hopper architecture, which introduces several key innovations for AI and HPC workloads. The architecture consists of several key components that work together to deliver exceptional performance.

1. Streaming Multiprocessors (SMs) - The Workers

Think of SMs as teams of workers in a factory. Each team can handle thousands of tasks at the same time. The H200 has many such teams working in parallel, making it incredibly fast.

Example: Like having 100 chefs cooking different dishes simultaneously in a restaurant kitchen, each SM handles different parts of the AI computation.

2. Tensor Cores - The Speed Boosters

These are special units that do math super fast. They can work with different levels of precision - like using rough calculations for speed or precise calculations for accuracy.

Example: Like a calculator that can switch between "quick estimate" mode (FP8) and "exact calculation" mode (FP32) depending on what you need.

3. HBM3e Memory - The Super Fast Storage

141GB of ultra-fast memory - that's like having 141,000 high-quality photos stored in memory that can be accessed instantly. The 4.8TB/s speed means it can transfer data equivalent to 1,000 high-definition movies per second!

4. NVLink - The Super Highway

When multiple GPUs work together, NVLink is like a super-fast highway connecting them. It's 900GB/s - imagine transferring 900 full-length movies between GPUs every second!

Virtualization: Sharing One GPU with Many

Imagine you have one powerful GPU, but multiple people or applications need to use it. Virtualization lets you split it up fairly, like sharing a pizza where everyone gets their guaranteed slice! The H200 supports two ways to do this: MIG and vGPU.

Real-World Example: Like an apartment building where one powerful generator (the GPU) can power multiple apartments (virtual machines), and each apartment gets guaranteed electricity without affecting others.

Multi-Instance GPU (MIG) - Permanent Partitions

MIG splits one GPU into up to 7 separate, isolated GPUs. Each gets its own guaranteed resources - like cutting a cake into slices where each slice is completely separate.

Example: Like a building with 7 separate apartments, each with its own guaranteed electricity, water, and space. One apartment can't affect another, and each knows exactly what resources it has.

How MIG Works - Like Dividing a Pizza

  • 1 slice (141GB): The whole pizza - maximum power for one big task
  • 2 slices (70GB each): Share between two applications
  • 3 slices (47GB each): Perfect for three different AI models
  • 7 slices (18GB each): Maximum sharing - 7 separate GPUs in one!
  • Each slice is completely isolated - like having separate kitchens
  • Guaranteed performance - you always know what you're getting

Virtual GPU (vGPU) - Flexible Sharing

vGPU lets multiple virtual machines share one physical GPU by taking turns. It's like time-sharing a car - different people use it at different times, and everyone gets fair access.

Example: Like a shared workspace where multiple people can use the same powerful computer, but they take turns. The system ensures everyone gets fair time and their work is completely separate.

Why vGPU is Useful

  • Share Resources: Multiple VMs can use one GPU - like carpooling!
  • Flexible Allocation: Give more memory to VMs that need it
  • Move VMs: Can move a VM to a different server without stopping it
  • Works Everywhere: Compatible with VMware, Citrix, and other systems
  • Fair Sharing: Everyone gets equal time - no one hogs the GPU
