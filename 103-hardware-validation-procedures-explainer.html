Hardware Validation Overview

Hardware validation ensures that GPU and DPU components function correctly before deployment in production environments. Validation procedures verify hardware integrity, performance characteristics, connectivity, and compatibility with system components. Proper validation prevents production issues, reduces downtime, and ensures reliable AI infrastructure operation. Validation workflows include power-on self-test (POST), functional testing, performance benchmarking, and stress testing.

GPU Hardware Validation

GPU validation procedures verify that GPU hardware is properly installed, detected by the system, and functioning correctly. Validation includes physical inspection, system detection, driver verification, and performance testing.

1Physical Inspection

Before power-on, perform visual inspection:

  • Verify GPU is properly seated in PCIe slot
  • Check PCIe power connectors are securely connected
  • Inspect for physical damage (scratches, bent pins, damaged connectors)
  • Verify cooling solution is properly installed (fans, heatsinks, liquid cooling)
  • Check GPU model matches expected hardware

2System Detection

Verify GPU is detected by the system BIOS/UEFI and operating system:

# Check PCIe device detection lspci | grep -i nvidia # Expected output shows GPU(s): # 01:00.0 3D controller: NVIDIA Corporation Device [GPU Model] # 02:00.0 3D controller: NVIDIA Corporation Device [GPU Model] # Check kernel messages for GPU detection dmesg | grep -i nvidia # Verify GPU appears in /proc/devices cat /proc/devices | grep nvidia

3Driver Verification

Verify GPU driver is installed and loaded:

# Check driver version and GPU detection nvidia-smi # Expected output shows: # Driver Version: 535.XX.XX # GPU 0: [GPU Name] (UUID: ...) # Memory: [Total] MiB # Compute Capability: X.X # Verify driver modules are loaded lsmod | grep nvidia # Check driver files exist ls -la /usr/lib/x86_64-linux-gnu/libnvidia* | head -5

4Basic Functionality Test

Test basic GPU functionality:

# Query GPU information nvidia-smi --query-gpu=name,driver_version,memory.total,power.max_limit --format=csv # Test GPU compute capability nvidia-smi --query-gpu=compute_cap --format=csv # Verify CUDA functionality (if CUDA toolkit installed) nvcc --version # Test GPU memory access nvidia-smi --query-gpu=memory.used,memory.free --format=csv

5Performance Benchmarking

Run performance benchmarks to verify GPU meets expected performance:

# Run bandwidth test (if available) # Using CUDA samples or custom benchmarks # Monitor GPU utilization during workload nvidia-smi dmon -s u -c 60 # Run matrix multiplication benchmark # (Example using cuBLAS or custom CUDA code) # Verify performance metrics: # - Memory bandwidth # - Compute throughput (FLOPS) # - PCIe bandwidth

Expected Performance Ranges:

  • A100: ~1.9 TFLOPS (FP64), ~312 TFLOPS (FP16 Tensor)
  • H100: ~67 TFLOPS (FP64), ~1000 TFLOPS (FP16 Tensor)
  • Memory bandwidth: A100 ~2TB/s, H100 ~3TB/s

6Stress Testing

Run stress tests to verify stability under load:

# Monitor GPU during extended workload # Run for 24+ hours to verify stability # Monitor temperature and power watch -n 1 nvidia-smi --query-gpu=index,temperature.gpu,power.draw,clocks.current.graphics --format=csv # Check for errors in system logs dmesg | grep -i error journalctl -k | grep -i nvidia # Verify no thermal throttling nvidia-smi --query-gpu=clocks.throttle_reasons.active --format=csv

Stress Test Criteria:

  • No system crashes or GPU resets
  • Temperature stays within operating limits
  • No memory errors or ECC corrections
  • Performance remains consistent
  • No driver errors in logs

DPU Hardware Validation

DPU validation procedures verify BlueField DPU hardware installation, firmware functionality, network connectivity, and storage acceleration capabilities. DPU validation is critical for ensuring proper offloading and acceleration functionality.

1Hardware Detection

Verify DPU is detected by the system:

# Check for DPU in PCIe devices lspci | grep -i mellanox # Expected output: # XX:XX.X Ethernet controller: Mellanox Technologies BlueField # Check for DPU network interfaces ip link show # Verify DPU appears in system ls -la /dev/mst/

2Firmware Verification

Verify DPU firmware version and status:

# Check firmware version mlxfwmanager # Query DPU firmware information flint -d /dev/mst/mt[device]_pciconf0 q # Verify firmware is compatible with DOCA version # Check NVIDIA documentation for compatibility matrix

3Network Interface Testing

Test DPU network interfaces:

# List DPU network interfaces ip link show # Test network connectivity ping -I [dpu_interface] [target_ip] # Test bandwidth iperf3 -c [target] -B [dpu_interface_ip] # Verify RDMA functionality (if applicable) ibstat ibping [target_lid]

4RShim Access Verification

Verify RShim interface for DPU management:

# Check RShim device ls -la /dev/rshim* # Test SSH access to DPU ssh root@[dpu_ip] # Verify DPU console access # Access via RShim console interface

5DOCA Functionality Test

Test DOCA runtime and applications:

# Verify DOCA runtime installation doca_version # Test DOCA services systemctl status doca # Run DOCA sample applications # (Examples: DOCA Flow, DOCA DPA, etc.) # Verify hardware acceleration # Test storage acceleration, network offloading, etc.

Power-On Self-Test (POST) Procedures

POST procedures verify hardware functionality during system boot. POST checks detect hardware failures before the operating system loads, enabling early problem identification.

BIOS/UEFI POST Checks

  • PCIe Enumeration: Verify all PCIe devices (GPUs, DPUs) are detected
  • Memory Check: Verify GPU memory is accessible
  • Power Check: Verify power delivery to GPUs/DPUs
  • Temperature Check: Verify initial temperature readings
  • Clock Verification: Verify GPU/DPU clocks initialize correctly

POST Failure Indicators

  • No Display: GPU not detected or failed initialization
  • Beep Codes: Hardware failure indicated by beep patterns
  • POST Error Messages: Specific error codes displayed during boot
  • System Hangs: System stops during POST, indicating hardware issue
  • LED Indicators: Status LEDs on GPUs/DPUs indicate failure states

POST Troubleshooting Steps

  1. Check power connections to GPUs/DPUs
  2. Verify PCIe slot functionality (try different slot)
  3. Check BIOS/UEFI settings for PCIe configuration
  4. Verify GPU/DPU is properly seated
  5. Check for BIOS/UEFI updates
  6. Test with minimal hardware configuration
  7. Review POST error codes in BIOS/UEFI logs

Diagnostic Tools and Procedures

Diagnostic tools help identify hardware issues and verify component functionality. Use appropriate diagnostic tools for comprehensive hardware validation.

GPU Diagnostic Tools

  • nvidia-smi: Query GPU status, temperature, power, utilization
  • nvidia-ml-py: Python API for GPU monitoring and diagnostics
  • NVIDIA Diagnostic Tools: Official NVIDIA diagnostic utilities
  • CUDA Samples: CUDA code samples for testing GPU functionality
  • Nsight Compute: GPU profiling and debugging tool

DPU Diagnostic Tools

  • mlxconfig: DPU configuration and query tool
  • flint: Firmware management and query tool
  • mst: Mellanox Software Tools for device management
  • ibstat/ibping: InfiniBand diagnostic tools
  • DOCA Tools: DOCA-specific diagnostic utilities

System Diagnostic Commands

# Check system logs for hardware errors dmesg | grep -i error journalctl -k | grep -i error # Check PCIe link status lspci -vv | grep -i "link width\|link speed" # Monitor system resources htop iostat -x 1 # Check temperature sensors sensors # Verify power supply status ipmitool sensor list | grep -i power

Validation Best Practices

Following best practices ensures thorough hardware validation and reliable production deployment.

Validation Workflow

  1. Pre-Installation: Verify hardware specifications match requirements
  2. Physical Inspection: Check for damage and proper packaging
  3. Installation: Follow proper installation procedures
  4. POST Verification: Verify hardware detection during boot
  5. Driver Installation: Install and verify drivers
  6. Functional Testing: Test basic functionality
  7. Performance Testing: Verify performance meets specifications
  8. Stress Testing: Test stability under load
  9. Documentation: Record validation results

Validation Documentation

Maintain comprehensive validation records:

  • Hardware serial numbers and model information
  • Firmware and driver versions
  • Test results and performance metrics
  • Issues encountered and resolutions
  • Validation date and technician information
  • Sign-off and approval

Quality Assurance

  • Use standardized validation procedures
  • Test in environment similar to production
  • Run validation for sufficient duration (24+ hours for stress tests)
  • Verify all components, not just primary functionality
  • Test failure scenarios and recovery procedures
  • Maintain validation equipment calibration
Validation Checklist: ✓ Physical inspection completed
✓ Hardware detected in BIOS/UEFI
✓ Drivers installed and verified
✓ Basic functionality tests passed
✓ Performance benchmarks meet specifications
✓ Stress tests completed (24+ hours)
✓ No errors in system logs
✓ Documentation completed
✓ Ready for production deployment
