DOCA SDK vs Runtime Overview

DOCA (Data Center Infrastructure-on-a-Chip Architecture) provides two distinct components: SDK (Software Development Kit) and Runtime. Understanding the difference between these components is crucial for proper DPU application development and deployment. The SDK is used during development to build applications, while the Runtime is deployed on production systems to run those applications. This separation enables efficient development workflows and optimized production deployments.

Development vs Deployment: DOCA SDK vs Runtime is like the difference between a construction site (SDK) and a finished building (Runtime). Developers use the SDK with all its tools, headers, and development libraries to build applications. Once built, applications run on systems with just the Runtime, which contains only the essential libraries and services needed for execution, without development tools.

When to Use SDK vs Runtime

Choosing between SDK and Runtime depends on the system's role: development systems need the SDK, while production systems only need the Runtime.

Use DOCA SDK When:

  • Developing Applications: Building new DPU applications
  • Compiling Code: Need to compile source code
  • Debugging: Need debugging tools
  • Learning: Learning DOCA development
  • Development Host: On development machines

Use DOCA Runtime When:

  • Running Applications: Executing DOCA applications
  • Production Deployment: Production DPU systems
  • Minimal Footprint: Need minimal installation
  • No Development: No development needed
  • Optimized Systems: Optimized production systems

Typical Workflow

  1. Development Host: Install SDK on development machine
  2. Develop: Develop and build application using SDK
  3. Build: Compile application into binary
  4. Deploy: Install Runtime on production DPU
  5. Run: Deploy and run application binary on DPU
Best Practice: Install DOCA SDK on development hosts where you build applications. Install DOCA Runtime on production DPU systems where applications run. This separation keeps production systems lean and optimized while providing full development capabilities on development systems. Applications built with SDK can run on systems with Runtime installed.
