Lustre Architecture Overview

Lustre is a parallel distributed file system designed for high-performance computing and AI workloads. It achieves exceptional performance through parallel I/O operations across multiple storage servers, enabling applications to access data at extremely high throughput rates. The system consists of several key components that work together to provide scalable, high-performance storage.

MDT: Metadata Target

The MDT (Metadata Target) stores file system metadata, including file locations, directory structure, permissions, and file attributes. It provides the information needed to locate and access files stored across OSTs.

File Locations

Function: Track file storage

Maintains mapping of files to their storage locations across OSTs.

Directory Structure

Function: Organize files

Manages hierarchical directory organization and file relationships.

Permissions & Attributes

Function: Access control

Stores file permissions, ownership, and metadata attributes.

MDT Functions

  • File Locations: Where each file is stored
  • Directory Structure: How files are organized
  • Permissions: Who can access what
  • Metadata: File attributes and properties

OSTs: Object Storage Targets

OSTs (Object Storage Targets) are storage servers that store file data. Each OST stores a portion of the file system's data, and multiple OSTs work in parallel to provide high throughput and scalable capacity.

Parallel Access

Parallel access enables applications to read and write data across multiple OSTs simultaneously. This parallel I/O pattern provides high aggregate throughput that scales with the number of OSTs, making Lustre ideal for high-performance computing workloads.
