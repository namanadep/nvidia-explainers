Initial Switch Setup

Perform initial setup of Spectrum switches including initialization, IP configuration, and basic settings. Initial setup establishes management connectivity and basic switch operation.

Switch Initialization

Initialize switch for first-time setup:

Initial Configuration Steps:

  1. Connect to switch console
  2. Complete initial setup wizard
  3. Set admin password
  4. Configure management IP
  5. Set hostname
  6. Configure time/NTP

Basic Configuration Commands:

# Connect to switch (via console or SSH) # Initial configuration commands # Set hostname switch(config)# hostname spectrum-switch-01 # Configure management IP switch(config)# interface mgmt0 switch(config-if)# ip address 192.168.1.10/24 switch(config-if)# exit # Set admin password switch(config)# username admin password [password] # Configure NTP switch(config)# ntp server [ntp-server]

Management Access

Configure management access:

SSH Configuration:

# Enable SSH switch(config)# feature ssh # Configure SSH access switch(config)# ssh key rsa 2048 switch(config)# ssh server enable

SNMP Configuration:

# Enable SNMP switch(config)# feature snmp # Configure SNMP community switch(config)# snmp-server community public ro switch(config)# snmp-server community private rw

VLAN Configuration

Configure VLANs (Virtual LANs) for network segmentation. VLAN configuration enables logical network separation and improves security and performance.

VLAN Creation

Create VLANs for network segmentation:

Create VLAN:

# Create VLAN switch(config)# vlan 100 switch(config-vlan)# name ai-training switch(config-vlan)# exit # Create multiple VLANs switch(config)# vlan 101 switch(config-vlan)# name ai-inference switch(config-vlan)# exit

Verify VLAN Configuration:

# Show VLANs switch# show vlan # Show VLAN details switch# show vlan id 100

Port Assignment

Assign ports to VLANs:

Access Port Configuration:

# Configure access port (single VLAN) switch(config)# interface ethernet 1/1 switch(config-if)# switchport mode access switch(config-if)# switchport access vlan 100 switch(config-if)# exit

Trunk Port Configuration:

# Configure trunk port (multiple VLANs) switch(config)# interface ethernet 1/10 switch(config-if)# switchport mode trunk switch(config-if)# switchport trunk allowed vlan 100,101,102 switch(config-if)# exit

QoS Configuration

Configure Quality of Service (QoS) for traffic prioritization and bandwidth management. QoS configuration ensures critical traffic gets priority and bandwidth limits are enforced.

QoS Traffic Classes

Configure QoS traffic classes:

QoS Configuration:

# Enable QoS switch(config)# qos # Configure traffic class switch(config)# class-map type qos match-all high-priority switch(config-cmap-qos)# match dscp 46 switch(config-cmap-qos)# exit # Configure policy map switch(config)# policy-map type qos ai-traffic switch(config-pmap-qos)# class high-priority switch(config-pmap-qos-class)# priority percent 50 switch(config-pmap-qos-class)# exit

Bandwidth Limits

Configure bandwidth limits for traffic classes:

Bandwidth Configuration:

# Configure bandwidth limits switch(config-pmap-qos)# class class-default switch(config-pmap-qos-class)# bandwidth percent 50 switch(config-pmap-qos-class)# exit switch(config-pmap-qos)# exit # Apply policy to interface switch(config)# interface ethernet 1/1 switch(config-if)# service-policy type qos input ai-traffic switch(config-if)# exit

Switch Management

Manage Spectrum switches through CLI, SNMP, and management interfaces. Effective management enables monitoring, configuration, and troubleshooting.

CLI Management

Access switch via CLI:

CLI Access:

# SSH to switch ssh admin@[switch-ip] # Access switch CLI # Commands are similar to Cisco NX-OS # Show switch information switch# show version switch# show inventory switch# show running-config

SNMP Management

Configure SNMP for monitoring:

SNMP Configuration:

# Enable SNMP switch(config)# feature snmp # Configure SNMP v2c switch(config)# snmp-server community public ro switch(config)# snmp-server community private rw # Configure SNMP v3 switch(config)# snmp-server user admin auth sha [password] priv aes-128 [password] switch(config)# snmp-server host [monitoring-server] version 3 auth admin

Switch Configuration Best Practices

Follow best practices for effective Spectrum switch configuration.

Configuration Best Practices

  • Document all configuration changes
  • Use consistent naming conventions
  • Backup configuration before changes
  • Test configuration in non-production
  • Use version control for configurations
  • Monitor switch performance
Switch Configuration Checklist: 1. Complete initial setup
2. Configure management IP
3. Set up SSH access
4. Configure VLANs
5. Assign ports to VLANs
6. Configure QoS
7. Set up SNMP monitoring
8. Backup configuration
9. Document configuration
10. Test and verify
