Node Categories and Groups Overview

Node categories and groups provide organizational structure for cluster management, enabling logical grouping of nodes based on characteristics, roles, or requirements. Categories define high-level node types (e.g., compute, storage, management), while groups organize nodes within categories for specific purposes (e.g., training nodes, inference nodes). This organization enables efficient resource allocation, policy management, and operational workflows. Understanding categories and groups helps design cluster organization strategies that simplify management and enable efficient operations.

Node Categories

Node categories define high-level node types based on function, hardware, or role. Categories enable logical organization and policy application at a high level.

Category Types

  • Function-Based: Compute, storage, management
  • Hardware-Based: GPU nodes, CPU nodes, storage nodes
  • Role-Based: Worker nodes, head nodes, login nodes
  • Environment-Based: Production, development, testing

Inheritance and Hierarchy

Category and group hierarchies enable inheritance of properties and policies, simplifying management and ensuring consistency.

Inheritance Model

  • Category Properties: Properties inherited by groups
  • Group Properties: Properties inherited by nodes
  • Policy Inheritance: Policies inherited down hierarchy
  • Override Capability: Override at lower levels

Hierarchy Benefits

  • Consistency: Consistent properties across hierarchy
  • Efficiency: Efficient policy management
  • Flexibility: Flexibility to override when needed
  • Simplification: Simplified management

Inheritance Examples

  • Category Policy: Category-level policy applies to all groups
  • Group Override: Group can override category policy
  • Node Override: Node can override group policy
  • Default Values: Default values from higher levels

Organization Strategies

Effective organization strategies help design category and group structures that simplify cluster management and enable efficient operations.

Organization Principles

  • Logical Grouping: Group nodes logically
  • Clear Purpose: Clear purpose for each category/group
  • Scalability: Design for scalability
  • Flexibility: Allow for future changes

Resource Allocation

  • Category Allocation: Allocate resources at category level
  • Group Allocation: Allocate resources at group level
  • Fair Sharing: Fair resource sharing
  • Priority Management: Manage priorities

Best Practices

  • Plan Structure: Plan category/group structure
  • Document Organization: Document organization
  • Review Regularly: Review and adjust as needed
  • Consistent Naming: Use consistent naming conventions
Organization Strategy: Organize clusters by creating categories for major node types (compute, storage, management), creating groups within categories for specific purposes (training, inference, development), and using inheritance to apply policies efficiently. Plan organization structure, document it clearly, and review regularly. Effective organization simplifies cluster management and enables efficient resource allocation and policy management.
